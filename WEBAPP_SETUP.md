# Почему не открывается страница в боте

Telegram открывает Web App **только по HTTPS**. Адрес `https://your-domain.com` в `.env` — это заглушка, по нему ничего не отдаётся.

## Что сделать

### 1. Локальная разработка (компьютер)

1. Запустите приложение (например на порту 8000):
   ```bash
   python3 run.py
   # или: uvicorn app.main:app --host 0.0.0.0 --port 8000
   ```

2. Поднимите туннель с HTTPS (например [ngrok](https://ngrok.com)):
   ```bash
   ngrok http 8000
   ```

3. В выводе ngrok будет строка вида:
   ```
   Forwarding   https://abc123.ngrok-free.app -> http://localhost:8000
   ```

4. В файле **`.env`** укажите этот HTTPS-адрес:
   ```
   WEBAPP_BASE_URL=https://abc123.ngrok-free.app
   ```

5. Перезапустите бота:
   ```bash
   python3 bot.py
   ```

После этого кнопка «Открыть веб-приложение» в боте будет открывать ваш сайт через ngrok.

### 2. Сервер с доменом

Разместите приложение на сервере с настроенным HTTPS (например через Nginx + Let's Encrypt). В `.env` укажите:

```
WEBAPP_BASE_URL=https://ваш-домен.com
```

---

**Важно:** в `.env` должен быть именно **HTTPS**-адрес, без `http://` и без `localhost`. И по этому адресу должен открываться ваш Matrix-фронтенд (тот же порт, на котором крутится FastAPI со статикой).
