# Matrix Marketing Simulator

Веб-приложение для симуляции матричного маркетинга: **90% в сеть (выплаты), 10% проекту**.

## Параметры

| Параметр | Значение |
|----------|----------|
| M1 | $10, доход $36, бонус за место $9 |
| M2 | $20, доход $72, бонус за место $18 |
| M3 | $40, доход $144, бонус за место $36 |
| M4 | $80, доход $288, бонус за место $72 |
| Комплект 4 матриц | $150 |
| При полном заполнении | $600 доход, $60 проекту, $540 в сеть, реинвест $150 → $390 чистыми |

Реинвест: M1→$10 (открывается M1), M2→$20 (M1,M2), M3→$40 (M1,M2,M3), M4→$80 (M1,M2,M3,M4).

## Запуск

```bash
cd matrix_marketing
pip install -r requirements.txt
python run.py
```

Откройте в браузере: **http://127.0.0.1:8000**

- Регистрация новых пользователей с выбором матриц и реферера
- Просмотр баланса, активных матриц, дерева рефералов
- Визуализация структуры матрицы (позиции 1–7)
- Ручное пополнение баланса для тестов
- Админ: проверка целостности, обработка пула ожидания, статистика

## Тестовый сценарий

```bash
python -m tests.test_scenario
```

Создаётся root с комплектом [1,2,3,4] и 8 пользователей под ним. После заполнения матриц root получает выплаты и реинвест; проверяется баланс ~$390 и наличие 4 активных матриц.

## API

- `POST /api/register` — регистрация (`username`, `referrer_id?`, `levels`)
- `GET /api/user/{id}`, `GET /api/user/{id}/matrices`, `GET /api/matrix/{id}`
- `GET /api/user/{id}/tree` — дерево рефералов
- `POST /api/user/{id}/purchase` — докупка матриц (`levels`)
- `POST /api/user/{id}/add-funds` — начисление на баланс (`amount`)
- `GET /api/events` — лог событий
- `GET /api/admin/verify`, `POST /api/admin/process-holding-pool`, `GET /api/stats`

Документация API: http://127.0.0.1:8000/docs
